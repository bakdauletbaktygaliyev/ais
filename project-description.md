# AIS — Architecture Insight System

## 📌 Общее описание

AIS (Architecture Insight System) — это платформа для глубокого понимания исходного кода программных проектов через визуализацию и интеллектуальный анализ.

Система превращает любой репозиторий в **навигабельную, интерактивную карту**, где разработчик может не просто читать код, а **исследовать его как структуру**.

AIS решает одну из самых сложных задач в разработке — **быстрое понимание чужого или сложного кода**.

---

## 🎯 Проблема

Современные проекты:

* содержат тысячи файлов
* имеют сложные зависимости
* плохо документированы
* трудно читаются целиком

Разработчики тратят часы и дни на:

* поиск нужной логики
* понимание архитектуры
* отслеживание связей между модулями

Классические инструменты:

* показывают код линейно
* не дают целостного понимания системы

---

## 💡 Решение

AIS представляет код как **иерархическую систему взаимосвязанных элементов**.

Вместо чтения файлов:

* пользователь видит структуру проекта
* понимает зависимости
* перемещается по уровням абстракции

Код становится:

> не текстом, а **пространством**, в котором можно ориентироваться

---

## 🧠 Основной принцип

AIS работает по принципу:

> **"От общего к частному"**

Пользователь начинает с общего вида проекта и постепенно углубляется:

```text
Репозиторий → Модули → Файлы → Внутренние элементы
```

Каждый уровень раскрывает новую детализацию, сохраняя контекст.

---

## 🔍 Навигация по коду

### 📊 Граф как основа

Проект отображается в виде графа:

* узлы — элементы системы
* связи — зависимости между ними

Граф позволяет:

* видеть структуру
* понимать связи
* находить ключевые компоненты

---

### 🔁 Погружение (Drill-Down)

Пользователь может:

* выбрать любой элемент
* перейти внутрь него
* увидеть его внутреннюю структуру

Это создает эффект:

> 🔍 «приближения» к коду

---

### 🔙 Обратная навигация

Пользователь может:

* вернуться на предыдущий уровень
* видеть путь (как breadcrumbs)
* не терять контекст

---

## 🧭 Ментальная модель

AIS превращает код в:

* карту (как Google Maps)
* граф (как сеть)
* структуру (как дерево)

Пользователь:

* исследует
* ориентируется
* принимает решения

---

## 🤖 Интеллектуальный помощник

В системе есть встроенный AI-ассистент, который:

* отвечает на вопросы о коде
* объясняет архитектуру
* помогает находить нужную логику

Пользователь может спросить:

* «Как работает этот модуль?»
* «Где происходит обработка платежей?»
* «Какие части системы связаны между собой?»

AI отвечает, опираясь на структуру проекта и его содержимое.

---

## 🧩 Взаимодействие пользователя

### Основной сценарий:

1. Пользователь вводит ссылку на проект
2. Система анализирует его
3. Отображается общий граф
4. Пользователь:

   * изучает структуру
   * кликает на элементы
   * углубляется внутрь
5. При необходимости задает вопросы AI
